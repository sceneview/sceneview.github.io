3D Rendering

Scene { }

Compose-native 3D scene

The Scene { } composable is the root of every 3D view. It manages the Filament rendering surface, camera, environment, and gesture handling automatically. Add nodes as composable children — state changes update the scene instantly.

Google Filament Compose-native Auto lifecycle 
@Composable
fun MyScene() {
    val engine = rememberEngine()
    val modelLoader = rememberModelLoader(engine)
    val envLoader = rememberEnvironmentLoader(engine)

    Scene(
        engine = engine,
        modelLoader = modelLoader,
        environment = rememberEnvironment(envLoader) {
            envLoader.createHDREnvironment("studio.hdr")!!
        },
        cameraManipulator = rememberCameraManipulator()
    ) {
        // Nodes go here
    }
}
ModelNode

glTF / GLB model loading

Load any .glb or .gltf model from assets with rememberModelInstance(). Returns null while loading — no crashes, no flash. Model animations play automatically with autoAnimate = true. Scale, rotate, translate — all driven by Compose state.

glTF / GLB PBR Materials Animations Auto-dispose 
Scene(engine = engine, modelLoader = modelLoader) {
    val model = rememberModelInstance(
        modelLoader, "models/car.glb"
    )
    model?.let {
        ModelNode(
            modelInstance = it,
            scaleToUnits = 1.0f,
            autoAnimate = true,
            animationLoop = true,
            isEditable = true // pinch + rotate
        )
    }
}
Geometry

Procedural geometry nodes

Built-in primitive shapes with full material support. No 3D modeling tool needed for simple scenes — compose geometry directly in Kotlin.

CubeNode SphereNode CylinderNode PlaneNode 
Scene(engine = engine, materialLoader = materialLoader) {
    CubeNode(
        engine = engine,
        size = Float3(0.5f, 0.5f, 0.5f),
        materialInstance = materialLoader
            .createColorInstance(Color.Blue)
    )
    SphereNode(
        engine = engine,
        radius = 0.3f,
        materialInstance = materialLoader
            .createColorInstance(Color.Red)
    )
}

Labels & UI in 3D

TextNode New

SDF-rendered 3D text anchored in world space. Supports font size, color, alignment, and billboard mode so labels always face the camera. Perfect for AR labels, annotations, and info overlays. 

TextNode(
    text = "Hello, World!",
    fontSize = 0.1f,
    color = Color(1f, 1f, 1f),
    isBillboard = true
)
BillboardNode New

Always-facing-camera quad. Load any texture or icon and it stays visible regardless of camera angle — ideal for waypoint markers, icons, and sprite-based indicators in AR. 

BillboardNode(
    engine = engine,
    materialInstance = materialLoader
        .createInstance("materials/icon.mat")
) {
    worldPosition = Float3(0f, 1.5f, 0f)
}

Atmosphere & Post-Processing

DynamicSkyNode

Time-of-day sky & sun

Physically accurate sky simulation with configurable time-of-day. The sun position, atmosphere scattering, and ambient lighting all update dynamically — animate from sunrise to sunset in real time.

Atmosphere scattering Dynamic lighting Animated time 
Scene(engine = engine) {
    DynamicSkyNode(
        engine = engine,
        timeOfDay = 14.5f, // 2:30 PM
        turbidity = 2.0f
    )
    ModelNode(modelInstance = model)
}

Physics

PhysicsNode

Real-time rigid body simulation

Add gravity, collisions, and forces to any node with PhysicsNode. Built on the Bullet Physics engine. Drop objects into a scene, throw them with a tap — physics handles the rest.

Bullet Physics Gravity & forces Collision shapes 
Scene(engine = engine) {
    // Static ground plane
    PhysicsNode(
        engine = engine,
        bodyType = BodyType.STATIC,
        shape = CollisionShape.Box(Float3(5f, 0.1f, 5f))
    )
    // Dynamic falling object
    PhysicsNode(
        engine = engine,
        bodyType = BodyType.DYNAMIC,
        shape = CollisionShape.Sphere(0.2f)
    ) {
        worldPosition = Float3(0f, 2f, 0f)
        ModelNode(modelInstance = ball)
    }
}
AR

Augmented Reality

ARScene { }

Tap-to-place in 15 lines

ARScene { } handles everything: camera permission, ARCore session init, plane detection, and the camera feed background. Just provide the onSingleTapConfirmed callback and the content block.

ARCore Plane detection Anchors
Full step-by-step codelab → 
var anchorNode by remember { mutableStateOf<AnchorNode?>(null) }
val model = rememberModelInstance(modelLoader, "chair.glb")

ARScene(
    engine = engine,
    modelLoader = modelLoader,
    planeRenderer = true,
    onSingleTapConfirmed = { hitResult ->
        anchorNode?.destroy()
        anchorNode = AnchorNode(
            engine, hitResult.createAnchor()
        )
    }
) {
    anchorNode?.let { anchor ->
        model?.let {
            ModelNode(modelInstance = it, scaleToUnits = 0.5f)
        }
    }
}
AnchorNode Real world

Locks 3D content to a real-world position. As ARCore refines its scene understanding, the anchor updates — keeping your model on the surface even as you move.

Augmented Images ARCore

Detect and track real-world 2D images — QR codes, posters, book covers — and anchor 3D content precisely on them using ARCore's image recognition.

Cloud Anchors Multi-user

Host AR anchors in the cloud and share them across devices. Build persistent multi-user AR experiences where all participants see the same 3D content.

Augmented Faces ARCore

Track 468 facial landmarks in real time. Apply 3D masks, accessories, or make-up overlays directly on a user's face with sub-millimetre accuracy.
